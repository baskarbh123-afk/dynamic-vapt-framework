## Finding #F-006 — ADFS OAuth Configuration Leaked via Authentication Redirect (gopps.global.com)

**Date Found:** 2026-03-12
**Tester:** Baskar
**Engagement:** Multi-Target Penetration Test

---

### 1. Overview

| Field | Details |
|---|---|
| **Vulnerability Type** | Sensitive Data Exposure |
| **Severity** | Medium |
| **CVSS Score** | 5.3 |
| **CVSS Vector** | CVSS:3.1/AV:N/AC:L/PR:N/UI:N/S:U/C:L/I:N/A:N |
| **OWASP Category** | A05:2021 – Security Misconfiguration |
| **Affected Component** | /api/security/authentication/login |
| **Authentication Required** | None |

---

### 2. Description

The authentication login endpoint at `/api/security/authentication/login` performs a 302 redirect that exposes the full ADFS OAuth2 configuration:

- **ADFS Server:** `adfs2.thisisglobal.com`
- **Client ID:** `gOpps.Prod`
- **Resource:** `urn:gOpps.Prod`
- **OAuth2 Flow:** Authorization Code
- **Redirect URI:** `https://gopps.global.com/api/security/authentication/token`

This reveals the identity provider infrastructure, the production client identifier, and the OAuth2 flow details, enabling targeted attacks against the authentication system.

---

### 3. Impact

**Business Impact:**
An attacker can use the leaked ADFS server address and client ID to craft OAuth2 phishing attacks, attempt token theft, or target the ADFS infrastructure directly.

**Technical Impact:**
- **Confidentiality:** Low — Authentication infrastructure details exposed
- **Integrity:** None
- **Availability:** None

---

### 4. Affected Endpoints / Parameters

| # | Method | Endpoint | Notes |
|---|---|---|---|
| 1 | GET | /api/security/authentication/login | 302 redirect leaks ADFS config |

---

### 5. Steps to Reproduce

```bash
curl -sI "https://gopps.global.com/api/security/authentication/login?redirectUri=https://gopps.global.com"
```

Observe the Location header contains the full ADFS OAuth2 authorization URL with client_id, resource, and redirect_uri parameters.

---

### 8. Remediation

- Do not expose OAuth client configuration details in client-visible redirects where possible
- Ensure the ADFS server is not directly accessible from the internet if not required
- Implement OAuth2 PKCE flow to prevent authorization code interception
