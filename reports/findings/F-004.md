### 2. Description

Multiple WordPress sites expose user information through the WP REST API `/wp-json/wp/v2/users` endpoint and the `?author=N` parameter. This allows unauthenticated attackers to enumerate usernames, user IDs, gravatar email hashes, and user profile URLs.

**Affected Targets and Exposed Users:**

**globalbuitenreclame.nl** (via /wp-json/wp/v2/users):
- User ID 1: "23G" (slug: 23g) — Admin account, URL reveals internal dev domain: `global-webshop.test/wp`
- User ID 3: "Martin Huijsbregts" (slug: martin)
- User ID 4: "Veronique lekkerkerker" (slug: veronique-lekkerkerker)
- All users have Gravatar hashes exposed (can be used for email discovery)

**fellasstudios.com** (via /wp-json/wp/v2/users):
- User ID 1: "admin3456" (slug: admin3456) — Admin username with numeric suffix exposed

**production.victoriousfestival.co.uk** (via /?author=N):
- Author 1: rmartin
- Author 3: mbishop
- Author 7: rob-jordan
- Author 8: bmiles
- Author 9: pcaruana

Note: www.victoriousfestival.co.uk and production.victoriousfestival.co.uk properly block /wp-json/wp/v2/users (403).

---

### 3. Impact

**Business Impact:**
Enumerated usernames can be used in targeted credential stuffing or brute-force attacks. The exposed admin username "admin3456" on fellasstudios.com is particularly concerning as it directly identifies the admin account. The internal development URL leak on globalbuitenreclame.nl reveals infrastructure details.

**Technical Impact:**
- **Confidentiality:** Low — Usernames, gravatar hashes, and internal URLs exposed
- **Integrity:** None
- **Availability:** None

1. Enumerate users on globalbuitenreclame.nl:
   ```bash
   curl -s "https://globalbuitenreclame.nl/wp-json/wp/v2/users" | python3 -m json.tool
   ```

2. Enumerate users on fellasstudios.com:


### 7. Root Cause Analysis

WordPress enables the REST API user listing by default. The `/wp-json/wp/v2/users` endpoint is publicly accessible unless explicitly restricted. Similarly, the `?author=N` parameter triggers a redirect that reveals usernames through the URL slug.

---

### 8. Remediation

**Primary Fix:**
Disable public access to the WP REST API users endpoint. Add to `functions.php`:
```php
add_filter('rest_endpoints', function($endpoints) {
    if (isset($endpoints['/wp/v2/users'])) {
        unset($endpoints['/wp/v2/users']);
    }
    if (isset($endpoints['/wp/v2/users/(?P<id>[\d]+)'])) {
        unset($endpoints['/wp/v2/users/(?P<id>[\d]+)']);
    }
    return $endpoints;
});
```

**Secondary Controls:**
- Disable author archives or use a security plugin to block author enumeration
- Use non-obvious admin usernames (not "admin3456")
- Remove internal development URLs from user profiles
- Implement login rate limiting and account lockout policies

---

### 9. Retest Criteria

This finding is remediated when:
- [ ] `/wp-json/wp/v2/users` returns 403 or 404 on all affected sites
- [ ] `/?author=N` does not reveal usernames via redirect
