## Finding #F-008 — WordPress REST API User Enumeration (blog.geizhals.at & unternehmen.geizhals.at)

**Date Found:** 2026-03-12
**Tester:** Baskar Mariyappan
**Engagement:** Geizhals.at Penetration Test
**Status:** DRAFT

---

### 1. Overview

| Field | Details |
|---|---|
| **Vulnerability Type** | Information Disclosure / User Enumeration |
| **Severity** | Medium |
| **CVSS Score** | 5.3 |
| **CVSS Vector** | CVSS:3.1/AV:N/AC:L/PR:N/UI:N/S:U/C:L/I:N/A:N |
| **OWASP Category** | A01:2021 – Broken Access Control |
| **Affected Component** | /wp-json/wp/v2/users |
| **Authentication Required** | None |

---

### 2. Description

Both WordPress sites under geizhals.at expose user account data via the REST API without authentication. The /wp-json/wp/v2/users endpoint returns user IDs, display names, slugs, author page URLs, and Gravatar email hashes for all registered users.

**blog.geizhals.at** exposes 10 users including: Alex (alexnebejea), Anjana (anjanamayrobnig), Barbara (anythingbuto), E.Berger (ephraimberger), Gast (bloggeizhals), Gerry (gerrgeizhals), GHGregor (zankarne), Kerstin (zuhausimaffenhaus), Khoa (geizhalskhoa), lisa (ghlisa).

**unternehmen.geizhals.at** exposes 3 users with full real names: Christian Peidelstein (christianpeidelstein), Michael Kröll (pierlopone), romangeizhals (romangeizhals).

---

### 3. Impact

**Business Impact:** Employee names and Gravatar hashes are exposed publicly. Gravatar hashes can potentially be reversed to recover email addresses. Enumerated usernames can be used in credential stuffing attacks.

**Technical Impact:**
- **Confidentiality:** Medium — User identities, slugs, and Gravatar email hashes exposed
- **Integrity:** None
- **Availability:** None

---

### 4. Steps to Reproduce

```
curl -s "https://blog.geizhals.at/wp-json/wp/v2/users" | python3 -m json.tool
curl -s "https://unternehmen.geizhals.at/wp-json/wp/v2/users" | python3 -m json.tool
```

---

### 5. Remediation

- Disable public access to the /wp-json/wp/v2/users endpoint
- Use a security plugin or functions.php filter to restrict the endpoint to authenticated users only
