## Finding #F-001 — Swagger API Documentation Publicly Exposed (gopps.global.com)

**Date Found:** 2026-03-12
**Tester:** Baskar
**Engagement:** Multi-Target Penetration Test

### 2. Description

The gOpps application (gopps.global.com) exposes its complete Swagger/OpenAPI documentation publicly without any authentication. The Swagger UI is accessible at `/swagger/ui/index` and the full API specification JSON is available at `/swagger/docs/v1`.

The exposed documentation reveals:
- **45 API endpoints** covering orders, brands, properties, slots, authentication, data export, and search
- **96 data models** including internal business objects (OrderAggregate, SlotAvailability, Property, BuyingArea, etc.)
- **Authentication mechanism details**: API key-based auth via `Authorization` header (Bearer token)
- **ADFS OAuth integration**: Auth endpoints at `/api/security/authentication/token` and `/api/security/authentication/login`
- **Data export endpoints**: `/api/data/slots/indexall`, `/api/data/orders/indexall`, `/api/data/orders/indexsincemodified`

This information significantly reduces the effort required for an attacker to map and exploit the application's API surface.

---

### 3. Impact

**Business Impact:**
An attacker can use the complete API documentation to craft targeted attacks against the order management, property, and slot management systems. The exposed endpoint structure, parameter names, and data models eliminate the enumeration phase of an attack.

**Technical Impact:**
- **Confidentiality:** Medium — Complete API structure, endpoint paths, parameter names, and data model schema exposed
- **Integrity:** None
- **Availability:** None

### 7. Root Cause Analysis

The Swagger/Swashbuckle middleware is deployed in production with default settings, allowing unauthenticated access to the API documentation endpoints. The `local: "true"` flag in `/api/about` suggests a development configuration may be running in production. Swagger documentation should be restricted to development environments or protected behind authentication.

---

### 8. Remediation

**Primary Fix:**
Disable Swagger UI and API documentation endpoints in production and staging environments. In ASP.NET with Swashbuckle, conditionally register Swagger only in development:

```csharp
if (env.IsDevelopment())
{
    app.UseSwagger();
    app.UseSwaggerUI(c => c.SwaggerEndpoint("/swagger/v1/swagger.json", "gOpps v1"));
}
```

**Secondary Controls:**
- Remove or restrict `/api/about` and `/api/status` endpoints in production
- Ensure the `local` flag is set to `false` in production configurations
- If API docs must be accessible, protect them behind authentication
- Review all environment-specific configurations to ensure development settings are not deployed to production
