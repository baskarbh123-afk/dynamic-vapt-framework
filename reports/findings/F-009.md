## Finding #F-009 — XMLRPC system.multicall Enabled (blog.geizhals.at & unternehmen.geizhals.at)

**Date Found:** 2026-03-12
**Tester:** Baskar Mariyappan
**Engagement:** Geizhals.at Penetration Test
**Status:** DRAFT

### 2. Description

The XMLRPC interface is enabled on both blog.geizhals.at and unternehmen.geizhals.at with 80 methods exposed, including system.multicall. This method allows bundling hundreds of authentication attempts into a single HTTP request, bypassing per-request rate limiting.

Additionally, pingback.ping is available which can be abused as an SSRF vector. Both sites are hosted on WordPress.com Atomic platform.

---

### 3. Impact

**Business Impact:** Brute-force amplification can bypass rate limiting controls. Combined with the user enumeration in F-008, attackers have valid targets for credential attacks.

**Technical Impact:**
- **Confidentiality:** Low — Enables credential brute-force
- **Integrity:** Low — pingback.ping SSRF risk
- **Availability:** None

---

### 4. Steps to Reproduce

```
curl -s -X POST "https://blog.geizhals.at/xmlrpc.php" -H "Content-Type: text/xml" \
  -d '<?xml version="1.0"?><methodCall><methodName>system.listMethods</methodName></methodCall>'
```

system.multicall confirmed functional:
```
curl -s -X POST "https://unternehmen.geizhals.at/xmlrpc.php" -H "Content-Type: text/xml" \
  -d '<?xml version="1.0"?><methodCall><methodName>system.multicall</methodName><params><param><value><array><data><value><struct><member><name>methodName</name><value><string>system.listMethods</string></value></member><member><name>params</name><value><array><data></data></array></value></member></struct></value></data></array></value></param></params></methodCall>'
```

---

### 5. Remediation

- Disable XMLRPC entirely via security plugin or WordPress.com settings
- If XMLRPC cannot be disabled (WordPress.com hosting), request Automattic to restrict system.multicall
- Implement WAF rules to block XMLRPC brute-force patterns
