### 2. Description

The production JavaScript bundle at `/static/js/main.dd4b9293.js` on gopps.global.com contains hardcoded references to internal infrastructure:

**Internal Environment URLs:**
- `https://goppsdev.global.com` (Development)
- `https://goppsint.global.com` (Integration)
- `https://goppstest.global.com` (Test)
- `https://goppsuat.global.com` (UAT — **publicly accessible**)
- `https://localhost:3000` (Local development)

**Authentication Infrastructure:**
- `https://adfs2.thisisglobal.com/adfs/oauth2/authorize` (ADFS OAuth endpoint)

**Third-party Services:**
- `https://frosty-dust-9852.getsandbox.com/api/` (Sandbox mock API)

The UAT environment (goppsuat.global.com) is confirmed to be publicly accessible and exposes the same Swagger API documentation as production.

---

### 3. Impact

**Business Impact:**
Exposed internal environment URLs allow attackers to target development, testing, and UAT environments which typically have weaker security controls. The ADFS endpoint reveals the identity provider infrastructure. The accessible UAT environment may contain test data and have relaxed authentication.

**Technical Impact:**
- **Confidentiality:** Medium — Internal infrastructure topology and authentication architecture exposed
- **Integrity:** None
- **Availability:** None

### 5. Steps to Reproduce

1. Download the production JavaScript bundle:
   ```bash
   curl -s "https://gopps.global.com/static/js/main.dd4b9293.js" > bundle.js
   ```

2. Search for environment URLs:
   ```bash
   grep -oE 'https://gopps[a-z]+\.global\.com' bundle.js | sort -u
   ```

3. Verify UAT environment is accessible:
   ```bash
   curl -sI "https://goppsuat.global.com"
   # Returns HTTP 200
   ```

4. Confirm Swagger is exposed on UAT:
   ```bash
   curl -s "https://goppsuat.global.com/swagger/ui/index" | head -5
   # Returns Swagger UI HTML
   ```

### 7. Root Cause Analysis

The React application's build process includes all environment configuration URLs in the production bundle rather than injecting only the production URL at build time. This is a common issue with React apps that use environment-based configuration without proper build-time variable substitution.

---

### 8. Remediation

**Primary Fix:**
- Use environment-specific build configurations to only include the production URL in production builds
- Remove all non-production URLs from the frontend code
- Use runtime environment variable injection instead of hardcoded URLs

**Secondary Controls:**
- Restrict UAT/dev/test environments to internal networks only (VPN/firewall)
- Remove Swagger from all non-development environments
- Implement network segmentation between environments

---

### 9. Retest Criteria

This finding is remediated when:
- [ ] Production JS bundle contains only the production URL
- [ ] Internal environment URLs are not accessible from the internet
- [ ] ADFS and sandbox API URLs are not present in client-side code
