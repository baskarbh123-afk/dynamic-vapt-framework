### 2. Description

The Next.js application at www.makesomenoise.com exposes its Sanity.io CMS configuration in the client-side HTML/RSC payload:

- **Project ID:** `lvz0au6x`
- **Dataset:** `production`
- **API Host:** `https://api.sanity.io`
- **API Endpoint:** `https://lvz0au6x.api.sanity.io`

The Sanity.io project ID and dataset name allow an attacker to query the public Sanity API directly to enumerate all published content, potentially including draft content if the dataset permissions are misconfigured.

---

### 3. Impact

**Business Impact:**
An attacker could query the Sanity API to enumerate all content, extract data structures, and potentially access unpublished/draft content if dataset permissions are not properly restricted.

**Technical Impact:**
- **Confidentiality:** Low — CMS project details exposed, potential content enumeration
- **Integrity:** None
- **Availability:** None

---

### 5. Steps to Reproduce

1. View page source of https://www.makesomenoise.com
2. Search for "sanity" or "projectId" in the RSC payload
3. Query the Sanity API:
   ```bash
   curl "https://lvz0au6x.api.sanity.io/v2021-10-21/data/query/production?query=*[_type%20%3D%3D%20%22page%22]"
   ```

---

### 8. Remediation

- Use server-side data fetching to avoid exposing the Sanity project ID in client-side code
- Restrict the Sanity API token permissions to read-only published content
- Enable Sanity's API CDN with token-gated access for sensitive datasets
