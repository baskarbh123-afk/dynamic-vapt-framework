## Finding #F-011 — User-Agent Reflected in Page Body (gct.geizhals.at)

**Date Found:** 2026-03-12
**Tester:** Baskar Mariyappan
**Engagement:** Geizhals.at Penetration Test
**Status:** DRAFT

| Field | Details |

### 2. Description

The Nextag price comparison engine at gct.geizhals.at reflects the HTTP User-Agent header directly into a hidden div element in the page body without sanitization. The element `<div id="userAgent">` contains the raw User-Agent value.

When tested with `<test>` as the User-Agent, the server rendered: `<div id="userAgent"><test></div>`

While the div is hidden via CSS (`display: none`), the unsanitized reflection of user input into HTML represents a content injection vulnerability. Although exploitation requires social engineering (victim must visit with a crafted User-Agent), the lack of output encoding indicates missing input sanitization practices.

Additionally, the page title and meta tags display raw i18n template keys (`home.seo.title`, `home.seo.description`, `home.seo.keyword`) instead of rendered content, indicating a misconfiguration in the internationalization framework.

The site also uses jQuery 1.12.0, which has multiple known CVEs including prototype pollution and XSS vulnerabilities.

---

### 3. Impact

**Business Impact:** Content injection in the page body. Combined with the outdated jQuery, this may have additional exploitation vectors.

**Technical Impact:**
- **Confidentiality:** None
- **Integrity:** Low — Unsanitized content injection
- **Availability:** None

---

### 4. Steps to Reproduce

```
curl -s "https://gct.geizhals.at/" -A '<test>' | grep 'userAgent'
```

Output: `<div id="userAgent"><test></div>`

i18n leak:
```
curl -s "https://gct.geizhals.at/" | grep -oE '<title>[^<]*</title>'
```

Output: `<title>home.seo.title</title>`

---

### 5. Remediation

- HTML-encode the User-Agent value before rendering in the DOM
- Fix the i18n configuration to render translated strings instead of template keys
- Upgrade jQuery from 1.12.0 to latest version (3.7+)
